# Contributing to Guixu

Thanks for helping improve Guixu.

## Before you open a PR

- Open an issue or start a discussion for large features, API changes, or breaking changes.
- Keep third-party code, data, models, images, and logos out of the repository unless you have verified that their licenses allow the intended use and redistribution.
- Add or update tests when your change affects behavior.
- Preserve the repository's SPDX headers and copyright notices on first-party files.

## Licensing and copyright

- Guixu is distributed under the Apache License, Version 2.0. See [LICENSE](LICENSE).
- The repository's attribution notices are collected in [NOTICE](NOTICE). Trademark rules are described in [TRADEMARKS.md](TRADEMARKS.md).
- Inbound contributions are accepted only under Apache-2.0 and are also subject to the contributor paperwork described in [CLA.md](CLA.md).
- For non-trivial contributions, maintainers may require a signed CLA before review or merge.
- The project currently designates `The State Key Laboratory of Blockchain and Data Security, Zhejiang University` as the project copyright holder for repository notices unless the maintainers publish an updated legal-entity notice.
- The project's default policy for substantive contributions is copyright assignment through the CLA. If your employer, sponsor, or university does not allow assignment, contact the maintainers before contributing; an alternative institutional agreement may be required and the contribution may be declined if rights are unclear.
- Pre-existing material that you keep ownership of must be clearly identified in the pull request and must be compatible with Apache-2.0 and the project's notice requirements.

## By submitting a contribution

Unless you explicitly state otherwise in writing and the maintainers agree, any Contribution intentionally submitted for inclusion in Guixu is provided under Apache-2.0. If you have signed the CLA, the CLA controls any additional assignment or retained-rights terms that apply to your contribution.

## Practical checklist

- Use clear commit messages and PR descriptions.
- Note any added dependencies and their licenses.
- Do not add third-party trademarks or logos in a way that implies endorsement.
- If you are contributing on behalf of an organization, make sure you have authority to do so.
- Run `python3 scripts/spdx_headers.py fix` if you add new first-party source or documentation files without the standard short copyright header.
